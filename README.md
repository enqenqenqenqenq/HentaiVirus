# SilentPackageManager

## 🔹 Описание проекта

**SilentPackageManager** — это специализированное WPF-приложение, выступающее в роли автоматизированного загрузчика и лаунчера. Программа функционирует исключительно на основе информированного согласия пользователя. После подтверждения программа инициализирует фоновые службы для асинхронной загрузки, многоуровневой распаковки и запуска программного обеспечения, управляя состоянием через локальную базу данных SQLite.

При повторном запуске в системе, где контент уже установлен, приложение предоставляет интерфейс для полного удаления скачанных файлов, завершения связанных процессов и очистки базы данных.

## 🔹 Технический стек

* **Фреймворк:** .NET 10.0 (WPF) 


* **Архитектура сборки:** `win-x64`, `SelfContained`, `PublishSingleFile` (с включением нативных библиотек).


* **Локальное хранилище:** SQLite (`Microsoft.Data.Sqlite`).
* **Работа с архивами:** `SharpCompress`, нативная библиотека `7z.dll` через `Squid-Box.SevenZipSharp`.
* **Сетевое взаимодействие:** `HttpClient` с кастомной обработкой Cookie и автоматическим обходом токенов подтверждения Google Drive (`confirm=t`).

## 🔹 Архитектура и основные компоненты

### 1. Движок загрузки и распаковки (`Downloader.cs`)

Отвечает за сетевые операции и обработку файлов.

* **Обход ограничений Google Drive:** Автоматически парсит HTML-ответы для извлечения токенов подтверждения (bypass large file warning).
* **Безопасная распаковка (Safe Extraction):** Использует паттерн постепенного понижения (fallback). Сначала пытается распаковать архив через управляемый API `IReader`, затем через `IArchive`. В случае неудачи (или специфичного формата) вызывает нативную `7z.dll`.
* **Умный поиск исполняемого файла:** Рекурсивно сканирует директорию после распаковки, фильтруя файлы-установщики (`setup`, `unins`, `vcredist`) и выбирая наиболее вероятный `.exe` файл по глубине расположения и размеру.

### 2. Менеджер базы данных (`DatabaseManager.cs`)

Управляет состоянием развёртывания (State Management) и хранит конфигурацию загрузок.

* **Таблица `Games`:** Хранит URL-адреса, целевые директории, пути к `.exe` и статус скачивания (`IsDownloaded`).
* **Изоляция данных:** Контент загружается в скрытую директорию `%APPDATA%\SilentPackageManager\Content`. Названия папок санитизируются для предотвращения ошибок файловой системы.
* **Полная очистка (`PurgeEverything`):** Модуль деинсталляции, который перед удалением файлов принудительно завершает (`Kill`) все связанные процессы, запущенные через лаунчер.

### 3. Пользовательский интерфейс и службы (`MainWindow.xaml.cs`)

UI выполняет роль консоли инициализации и мониторинга.

* **Фоновые циклы:** После принятия соглашения  запускаются два независимых асинхронных цикла:


* `DownloadEngineAsync`: проверяет базу данных на наличие нескачанных ссылок (интервал 2 минуты).
* `LaunchEngineAsync`: циклично запускает готовые приложения (интервал 1 минута).


* **Потокобезопасное логирование:** Подписка на события `AppLogger` с диспетчеризацией в UI-поток для обновления текстового поля в реальном времени.

## 🔹 Инструкция по сборке и развёртыванию

1. **Зависимости:** Убедитесь, что установлен .NET 10.0 SDK.
2. **Нативные библиотеки:** В корне проекта должен присутствовать файл `7z.dll`. В конфигурации проекта (`.csproj`) настроено его копирование в выходную директорию (`PreserveNewest`).
3. **Публикация:** Для создания единого исполняемого файла используйте команду:
```bash
dotnet publish -c Release -r win-x64 --self-contained true

```


4. **Первоначальная настройка:** При первом запуске база данных автоматически инициализируется (`SeedDatabaseWithLinks`) заранее определёнными ссылками на Google Drive.
